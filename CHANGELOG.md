# Changelog

## 2026-04-14

### 알림 권한 / 설정
- Android 13 이상은 시스템 알림 권한 요청을 사용하고, Android 12 이하는 시스템 팝업 대신 앱 자체 안내 다이얼로그로 내부 알림 설정을 켜도록 분리했습니다.
- 첫 설치 기본 알림 설정을 `OFF`로 두고, 사용자가 동의한 뒤에만 앱 내부 알림 설정이 켜지도록 `notification_consent_asked` 저장값을 추가했습니다.
- Android SDK 버전을 Flutter에서 확인할 수 있도록 네이티브 `MethodChannel`을 추가했습니다.
- 설정 화면에서 저장되는 알림 설정이 새 동의 상태 값을 유지하도록 정리했습니다.

### 알림 라우팅 / 최근 알림
- 알림 메시지를 눌러 쿠폰 상세로 이동한 뒤 앱을 종료하고 아이콘으로 재실행했을 때, 직전 알림 상세로 다시 이동하지 않고 홈으로 시작하도록 라우팅 상태 초기화 흐름을 추가했습니다.
- 알림 리스트의 시간 표시가 24시간 경과 기준이 아니라 날짜 경계 기준으로 `1일 전`처럼 표시되도록 보정했습니다.

### 웹 / Chrome 실행
- Chrome 실행 시 `flutter_local_notifications` 웹 미지원 호출로 앱이 멈추지 않도록 웹 환경에서는 알림 플러그인 호출을 건너뛰도록 처리했습니다.
- Chrome SQLite 실행 오류를 줄이기 위해 `app/web/sqlite3.wasm`을 현재 `sqlite3` 패키지와 맞는 바이너리로 교체했습니다.

### 공통 UI
- 쿠폰/멤버십/알림 리스트와 상세 바텀시트의 empty 상태에서 `assets/icon/4.png` 기반 공통 마스코트 empty UI를 사용하도록 정리했습니다.
- `Icons.arrow_back` 기반 커스텀 뒤로가기 버튼들의 좌측 여백을 줄여 화면 가장자리와 더 자연스럽게 맞췄습니다.
- 쿠폰 검색 필드에서 내부 `TextField`가 바깥 `Container`의 라운드 border를 침범하지 않도록 높이와 폭을 보정하고, 보정 이유를 코드 주석으로 남겼습니다.

### iOS / 빌드 설정
- iOS Pod sandbox 불일치 상태를 `pod install`로 정리하고, archive 버전이 `0.1.2 (4)`를 바라보도록 Flutter 생성 설정을 다시 맞췄습니다.

### 검증
- Android 실기기 `SM F946N`에서 디버그 실행을 확인했습니다.
- Chrome 디버그 실행을 확인했습니다.
- 변경 파일 단위 `flutter analyze`를 통과했습니다.

## 2026-04-13

### 알림 / 최근 알림 인박스
- 최근 알림 화면이 진입 시점뿐 아니라 앱 복귀 및 쿠폰 상세 화면에서 돌아온 뒤에도 알림 로그를 다시 불러오도록 수정했습니다.
- 알림 재스케줄 과정에서 `notification_logs` 전체가 삭제되어 최근 알림 기록이 사라지던 문제를 수정했습니다.
- 알림을 하나 눌렀을 때 전체 예약 알림/기록이 함께 사라질 수 있던 흐름을 줄이기 위해 재스케줄과 알림 기록 삭제를 분리했습니다.
- 알림 payload를 `couponId|type` 형식으로 확장해 어떤 알림 타입을 눌렀는지 구분하고, 선택한 알림만 읽음 처리되도록 개선했습니다.
- 기존 `couponId` 단독 payload도 계속 처리되도록 하위 호환을 유지했습니다.
- 앱이 켜져 있는 상태에서도 테스트 알림을 확인할 수 있도록 설정 화면의 테스트 알림 흐름에 foreground 보조 타이머를 추가했습니다.
- 테스트 알림용 예약 알림을 별도로 취소할 수 있는 흐름을 추가해 foreground 테스트와 OS 예약 테스트가 중복되지 않도록 정리했습니다.
- 최근 알림 카드에서 D-day 배지와 알림 제목을 가로 배치에서 세로 Column 배치로 변경했습니다.

### 권한 / 스토어 제출 준비
- iOS에서 실시간 카메라 스캔을 사용하지 않는 현재 기능 범위에 맞춰 `NSCameraUsageDescription`, `NSBonjourServices`, `NSLocalNetworkUsageDescription`를 제거했습니다.
- 앱 설명에서 `바코드/QR 실시간 스캔` 표현을 `갤러리 이미지 기반 바코드/QR 인식`으로 정리했습니다.
- Android의 카메라 권한과 라이브 카메라 스캔 화면을 제거하고, 갤러리 이미지 기반 OCR/바코드/QR 인식 흐름으로 정리했습니다.
- `mobile_scanner` 의존성을 제거하고 갤러리 이미지 분석용 `google_mlkit_barcode_scanning` 기반 흐름을 유지했습니다.

### Android 배포 / 버전
- Android 내부 테스트용 AAB 생성을 위해 release signing 설정을 추가했습니다.
- 실제 `key.properties` 대신 `key.properties.example` 문서를 추가해 keystore 설정 방법을 안내했습니다.
- release 빌드에서 ProGuard/R8 설정과 ML Kit 관련 경고 완화 규칙을 추가했습니다.
- 설정 화면 하단에 현재 앱 버전명과 빌드 번호를 눈에 띄지 않게 표시하도록 `package_info_plus`를 연결했습니다.
- 앱 버전을 `0.1.2+4` 기준으로 갱신했습니다.

### 공통 품질
- Flutter 최신 API 기준에 맞춰 일부 `withOpacity` 사용을 `withValues`로 정리하고 `flutter analyze`가 통과하도록 정리했습니다.

- `flutter clean`, `flutter pub get`, `pod install`을 다시 진행해 iOS archive 전 빌드 산출물과 Flutter 생성 설정을 재정리했습니다.
- iOS `Generated.xcconfig`가 누락되어 archive가 막히던 상태를 복구했습니다.
- iOS AppIcon 세트를 다시 생성해 App Store 업로드에서 요구하는 불투명 아이콘 기준에 맞게 정리했습니다.
- TestFlight 업로드 전 확인용으로 Bundle ID, App Store Connect 입력값, archive 절차를 문서화했습니다.
- `sqflite_common_ffi` / `sqlite3` 조합으로 iOS 빌드가 깨지던 문제를 정리하고 `flutter build ios --no-codesign` 기준 실제 빌드가 다시 통과하도록 복구했습니다.
- App Store Connect 업로드 이후 TestFlight 내부 테스트 설치까지 완료했습니다.

## 2026-04-06

### 데이터 / 저장 구조
- 쿠폰, 멤버십, 알림 설정을 가상 데이터가 아닌 실제 로컬 SQLite 데이터로 저장하도록 전환했습니다.
- 이미지 메타데이터와 로컬 파일 참조를 분리한 `image_assets` 구조를 추가해, 등록한 이미지가 앱 재실행 후에도 유지되도록 정리했습니다.
- 알림 리스트가 쿠폰 상태 기반 가상 목록이 아니라 `notification_logs` 기반의 실제 알림 인박스로 동작하도록 바꿨습니다.
- 로컬 데이터 구조 문서로 ERD, 데이터셋 정의, SQL 스키마 문서를 추가했습니다.

### 알림 / 로컬 알림
- `flutter_local_notifications` 기반 로컬 알림 서비스를 추가해 쿠폰 만료 전 알림을 등록, 취소, 재등록할 수 있도록 구현했습니다.
- 알림 클릭 시 홈을 거쳐 해당 쿠폰 상세 화면으로 이동하는 라우팅을 연결했습니다.
- 설정 화면 알림 토글과 실제 알림 스케줄이 연동되도록 정리했습니다.
- 읽은 알림은 흐린 카드 + 삭제 버튼이 보이고, 안 읽은 알림은 화살표가 보이는 구조로 정리했습니다.

### 웹 / Chrome 대응
- Chrome에서 SQLite 초기화가 실패하던 문제를 줄이기 위해 `sqflite_common_ffi_web` 기반 초기화 흐름을 정리했습니다.
- 웹 실행에 필요한 `sqlite3.wasm` 자산을 `app/web`에 추가하고, 현재 패키지 버전에 맞는 바이너리로 교체했습니다.

### 권한 / 시작 흐름
- 앱 첫 실행 시 스플래시에서 커스텀 알림 안내 다이얼로그와 시스템 알림 권한 요청이 겹치지 않도록 조정했습니다.
- 앱 첫 실행에서는 시스템 알림 권한 요청만 진행되도록 바꾸고, 사진 권한은 실제 이미지 추가/수정 시점에만 요청하도록 정리했습니다.

### 쿠폰 리스트 / 홈
- 쿠폰 리스트 항목 썸네일이 실제 저장된 쿠폰 이미지(`imageBytes`/로컬 파일)를 표시하도록 수정했습니다.
- 쿠폰 검색이 쿠폰명뿐 아니라 브랜드, 카테고리까지 함께 검색하도록 확장했습니다.
- 검색 입력 UI의 뒤 배경이 어색하게 보이던 부분을 정리해 더 깔끔한 검색 필드 스타일로 다듬었습니다.
- `SavingSpeechBubbleCard`가 이번 달에 실제로 사용 완료된 쿠폰들의 예상 금액 합계를 계산해 표시하도록 바꿨습니다.
- `TopMascotHeader`를 탭하면 절약 말풍선 카드 문구가 순환되도록 만들고, 기존 문구 외에 센스 있는 추가 문구 2개 이상을 더했습니다.

### 쿠폰 상세 / 수정 / 등록
- 쿠폰 상세 화면의 멤버십 리스트 바텀시트에서 항목을 탭하면 실제 해당 멤버십 상세 화면으로 이동하도록 수정했습니다.
- 쿠폰 상세 화면에서 이미지를 눌러 여는 전체화면 팝업의 닫기 버튼을 상단바 느낌으로 더 잘 보이게 조정했습니다.
- 쿠폰 상세 화면에서 `수정하기`로 이동했을 때 기존 이미지가 수정 화면에 보이지 않던 문제를 고쳤습니다.
- 쿠폰 수정 화면에서는 하단 CTA가 `추가하기`가 아니라 `수정하기`로 보이도록 수정했습니다.
- 쿠폰 상세 화면에서 이미 `사용 완료됨` 상태인 쿠폰 버튼을 다시 누르면 `사용 미완료 처리` 확인 팝업이 뜨고, 확인 시 다시 사용 가능 상태로 되돌리도록 구현했습니다.
- 쿠폰 등록/수정 화면에서 이미지가 추가된 상태일 때 이미지를 크게 볼 수 있는 `이미지 보기` 버튼을 오버레이로 추가했습니다.

### 멤버십 상세 / 수정 / 등록
- 멤버십 상세 화면의 쿠폰 리스트 바텀시트에서 항목을 탭하면 실제 해당 쿠폰 상세 화면으로 이동하도록 수정했습니다.
- 멤버십 상세 화면에서 이미지를 눌러 여는 전체화면 팝업의 닫기 버튼을 상단바 느낌으로 더 잘 보이게 조정했습니다.
- 멤버십 상세 화면에서 바코드 영역을 탭했을 때 전체화면 바코드 팝업이 뜨도록 `_showBarcodeFullScreen` 흐름을 구현했습니다.
- 멤버십 등록/수정 화면에서도 이미지가 추가된 상태일 때 이미지를 크게 볼 수 있는 `이미지 보기` 버튼을 추가했습니다.
- 멤버십 등록/수정 화면에 `이미지에서 정보 추출하기` 버튼을 추가해 이미지 속 바코드/텍스트를 읽어 카드 번호와 이름을 자동으로 채울 수 있게 했습니다.

### 알림 / 설정
- 알림 리스트에서 알림이 없을 때 `전체 삭제` 버튼이 비활성 상태로 보이도록 수정했습니다.
- 설정 화면의 테스트 알림 버튼을 단일 알림이 아니라 `30일 전 / 7일 전 / 3일 전 / 1일 전 / 당일 / 만료` 전 케이스를 한 번에 테스트하는 버튼으로 변경했습니다.
