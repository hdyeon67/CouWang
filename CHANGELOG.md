# Changelog

## 2026-04-07

### iOS / TestFlight 준비
- 설정 화면이 하단 탭 루트로 열렸을 때 뒤로가기 버튼이 빈 네비게이터를 `pop()` 하며 빨간 화면이 뜨던 문제를 수정했습니다.
- iOS Podfile의 `use_frameworks!` 구성을 `static linkage` 기준으로 정리하고 `pod install`을 다시 수행했습니다.
- `flutter clean`, `flutter pub get`, `pod install`을 다시 진행해 iOS archive 전 빌드 산출물과 Flutter 생성 설정을 재정리했습니다.
- iOS `Generated.xcconfig`가 누락되어 archive가 막히던 상태를 복구했습니다.
- iOS AppIcon 세트를 다시 생성해 App Store 업로드에서 요구하는 불투명 아이콘 기준에 맞게 정리했습니다.
- TestFlight 업로드 전 확인용으로 Bundle ID, App Store Connect 입력값, archive 절차를 문서화했습니다.
- `sqflite_common_ffi` / `sqlite3` 조합으로 iOS 빌드가 깨지던 문제를 정리하고 `flutter build ios --no-codesign` 기준 실제 빌드가 다시 통과하도록 복구했습니다.
- App Store Connect 업로드 이후 TestFlight 내부 테스트 설치까지 완료했습니다.

## 2026-04-06

### 데이터 / 저장 구조
- 쿠폰, 멤버십, 알림 설정을 가상 데이터가 아닌 실제 로컬 SQLite 데이터로 저장하도록 전환했습니다.
- 이미지 메타데이터와 로컬 파일 참조를 분리한 `image_assets` 구조를 추가해, 등록한 이미지가 앱 재실행 후에도 유지되도록 정리했습니다.
- 알림 리스트가 쿠폰 상태 기반 가상 목록이 아니라 `notification_logs` 기반의 실제 알림 인박스로 동작하도록 바꿨습니다.
- 로컬 데이터 구조 문서로 ERD, 데이터셋 정의, SQL 스키마 문서를 추가했습니다.

### 알림 / 로컬 알림
- `flutter_local_notifications` 기반 로컬 알림 서비스를 추가해 쿠폰 만료 전 알림을 등록, 취소, 재등록할 수 있도록 구현했습니다.
- 알림 클릭 시 홈을 거쳐 해당 쿠폰 상세 화면으로 이동하는 라우팅을 연결했습니다.
- 설정 화면 알림 토글과 실제 알림 스케줄이 연동되도록 정리했습니다.
- 읽은 알림은 흐린 카드 + 삭제 버튼이 보이고, 안 읽은 알림은 화살표가 보이는 구조로 정리했습니다.

### 웹 / Chrome 대응
- Chrome에서 SQLite 초기화가 실패하던 문제를 줄이기 위해 `sqflite_common_ffi_web` 기반 초기화 흐름을 정리했습니다.
- 웹 실행에 필요한 `sqlite3.wasm` 자산을 `app/web`에 추가하고, 현재 패키지 버전에 맞는 바이너리로 교체했습니다.

### 권한 / 시작 흐름
- 앱 첫 실행 시 스플래시에서 커스텀 알림 안내 다이얼로그와 시스템 알림 권한 요청이 겹치지 않도록 조정했습니다.
- 앱 첫 실행에서는 시스템 알림 권한 요청만 진행되도록 바꾸고, 사진 권한은 실제 이미지 추가/수정 시점에만 요청하도록 정리했습니다.

### 쿠폰 리스트 / 홈
- 쿠폰 리스트 항목 썸네일이 실제 저장된 쿠폰 이미지(`imageBytes`/로컬 파일)를 표시하도록 수정했습니다.
- 쿠폰 검색이 쿠폰명뿐 아니라 브랜드, 카테고리까지 함께 검색하도록 확장했습니다.
- 검색 입력 UI의 뒤 배경이 어색하게 보이던 부분을 정리해 더 깔끔한 검색 필드 스타일로 다듬었습니다.
- `SavingSpeechBubbleCard`가 이번 달에 실제로 사용 완료된 쿠폰들의 예상 금액 합계를 계산해 표시하도록 바꿨습니다.
- `TopMascotHeader`를 탭하면 절약 말풍선 카드 문구가 순환되도록 만들고, 기존 문구 외에 센스 있는 추가 문구 2개 이상을 더했습니다.

### 쿠폰 상세 / 수정 / 등록
- 쿠폰 상세 화면의 멤버십 리스트 바텀시트에서 항목을 탭하면 실제 해당 멤버십 상세 화면으로 이동하도록 수정했습니다.
- 쿠폰 상세 화면에서 이미지를 눌러 여는 전체화면 팝업의 닫기 버튼을 상단바 느낌으로 더 잘 보이게 조정했습니다.
- 쿠폰 상세 화면에서 `수정하기`로 이동했을 때 기존 이미지가 수정 화면에 보이지 않던 문제를 고쳤습니다.
- 쿠폰 수정 화면에서는 하단 CTA가 `추가하기`가 아니라 `수정하기`로 보이도록 수정했습니다.
- 쿠폰 상세 화면에서 이미 `사용 완료됨` 상태인 쿠폰 버튼을 다시 누르면 `사용 미완료 처리` 확인 팝업이 뜨고, 확인 시 다시 사용 가능 상태로 되돌리도록 구현했습니다.
- 쿠폰 등록/수정 화면에서 이미지가 추가된 상태일 때 이미지를 크게 볼 수 있는 `이미지 보기` 버튼을 오버레이로 추가했습니다.

### 멤버십 상세 / 수정 / 등록
- 멤버십 상세 화면의 쿠폰 리스트 바텀시트에서 항목을 탭하면 실제 해당 쿠폰 상세 화면으로 이동하도록 수정했습니다.
- 멤버십 상세 화면에서 이미지를 눌러 여는 전체화면 팝업의 닫기 버튼을 상단바 느낌으로 더 잘 보이게 조정했습니다.
- 멤버십 상세 화면에서 바코드 영역을 탭했을 때 전체화면 바코드 팝업이 뜨도록 `_showBarcodeFullScreen` 흐름을 구현했습니다.
- 멤버십 등록/수정 화면에서도 이미지가 추가된 상태일 때 이미지를 크게 볼 수 있는 `이미지 보기` 버튼을 추가했습니다.
- 멤버십 등록/수정 화면에 `이미지에서 정보 추출하기` 버튼을 추가해 이미지 속 바코드/텍스트를 읽어 카드 번호와 이름을 자동으로 채울 수 있게 했습니다.

### 알림 / 설정
- 알림 리스트에서 알림이 없을 때 `전체 삭제` 버튼이 비활성 상태로 보이도록 수정했습니다.
- 설정 화면의 테스트 알림 버튼을 단일 알림이 아니라 `30일 전 / 7일 전 / 3일 전 / 1일 전 / 당일 / 만료` 전 케이스를 한 번에 테스트하는 버튼으로 변경했습니다.

### 멤버십 리스트
- 멤버십 리스트의 `onMenuTap`은 추후 수정/삭제 같은 빠른 관리 액션을 담는 메뉴 용도임을 화면 메시지 기준으로 더 명확히 정리했습니다.
