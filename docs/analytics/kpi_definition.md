# 쿠왕(CouWang) KPI 정의

## 1. 프로젝트 핵심 목표
쿠왕 MVP의 핵심 목표는 사용자가 등록한 쿠폰을 잊지 않고, 만료 전에 확인하여 실제 사용 완료까지 이어지게 만드는 것이다.  
따라서 이번 프로젝트에서는 단순 등록 수보다 "등록된 쿠폰이 실제로 사용되었는가"를 가장 중요한 성과로 본다.

## 2. Primary KPI

### 2-1. 사용 완료율
- 정의: 등록된 쿠폰 중 실제로 사용 완료 처리된 쿠폰의 비율
- 목적: 쿠왕이 단순 보관 도구가 아니라 실제 사용을 유도했는지 판단하기 위한 핵심 지표

공식:

`사용 완료율 = 사용 완료 쿠폰 수 / 등록 완료 쿠폰 수`

| 항목 | 정의 |
|---|---|
| KPI명 | 사용 완료율 |
| 분자 | `coupon__complete__redeem` 이벤트가 발생한 쿠폰 수 |
| 분모 | `coupon__complete__create` 이벤트가 발생한 쿠폰 수 |
| 기본 기간 | 일/주 단위 집계 가능, MVP에서는 전체 기간 기준을 우선 사용 |
| 해석 | 높을수록 등록된 쿠폰이 실제 사용으로 이어졌음을 의미 |

## 3. Secondary KPI

### 3-1. 등록 완료율
- 정의: 등록 시작 대비 등록 완료 비율
- 목적: 쿠폰 등록 플로우가 너무 복잡하지 않은지 확인

`등록 완료율 = 등록 완료 수 / 등록 시작 수`

| 항목 | 정의 |
|---|---|
| 분자 | `coupon__complete__create` 이벤트 수 |
| 분모 | `coupon__start__create` 이벤트 수 |
| 기간 | 일/주/전체 기간 |

### 3-2. 만료율
- 정의: 등록된 쿠폰 중 사용되지 못하고 만료된 쿠폰의 비율
- 목적: 쿠왕이 "쿠폰을 놓치지 않게 하는 서비스"라는 목표를 얼마나 달성했는지 확인

`만료율 = 만료 완료 쿠폰 수 / 등록 완료 쿠폰 수`

| 항목 | 정의 |
|---|---|
| 분자 | `coupon__complete__expire` 이벤트가 발생한 쿠폰 수 |
| 분모 | `coupon__complete__create` 이벤트가 발생한 쿠폰 수 |
| 기간 | 일/주/전체 기간 |

### 3-3. 알림 오픈율
- 정의: 발송된 만료 임박 알림 중 사용자가 실제로 연 비율
- 목적: 알림이 사용자의 재방문과 확인 행동을 유도하는지 확인

`알림 오픈율 = 알림 오픈 수 / 알림 발송 수`

| 항목 | 정의 |
|---|---|
| 분자 | `notification__open__expiry_reminder` 이벤트 수 |
| 분모 | `notification__send__expiry_reminder` 이벤트 수 |
| 기간 | 알림 발송일 기준 일/주/전체 기간 |

## 4. Guardrail 지표
Guardrail 지표는 Primary KPI를 높이는 과정에서 서비스 경험이 나빠지지 않는지 확인하기 위한 보조 지표다.

### 4-1. 상세 조회율
- 정의: 등록된 쿠폰 중 상세 조회까지 이어진 비율
- 목적: 사용자가 등록 후 다시 쿠폰을 확인하고 관리하는 행동이 있는지 확인

`상세 조회율 = 상세 조회 쿠폰 수 / 등록 완료 쿠폰 수`

### 4-2. 알림 대비 과도한 만료율
- 정의: 알림이 발송되었음에도 만료로 끝난 쿠폰의 비율
- 목적: 알림 정책이 실질적으로 효과가 있는지 점검

`알림 후 만료율 = 알림 발송된 쿠폰 중 만료된 쿠폰 수 / 알림 발송된 쿠폰 수`

### 4-3. 미조회 사용 완료 비율
- 정의: 상세 조회 없이 바로 사용 완료된 쿠폰 비율
- 목적: 퍼널 정의와 실제 행동 차이를 확인하고 해석 오류를 줄이기 위함

## 5. 퍼널 정의
쿠왕 MVP의 핵심 퍼널은 아래 순서를 따른다.

1. `coupon__start__create`
2. `coupon__complete__create`
3. `coupon__view__detail`
4. `coupon__complete__redeem`

설명:
- 등록 시작: 사용자가 쿠폰 등록을 시도한 상태
- 등록 완료: 쿠폰이 저장된 상태
- 상세 조회: 저장된 쿠폰을 다시 열어본 상태
- 사용 완료: 실제 사용 처리까지 완료한 상태

주의:
- 모든 사용 완료가 반드시 상세 조회를 거친다고 가정하지는 않는다.
- 따라서 퍼널은 대표 흐름으로 사용하고, 실제 분석에서는 단계 간 누락 가능성을 함께 본다.

## 6. 각 KPI의 분모/분자/기간 정의

| KPI명 | 분자 | 분모 | 기본 기간 |
|---|---|---|---|
| 사용 완료율 | 사용 완료 쿠폰 수 | 등록 완료 쿠폰 수 | 전체/주간 |
| 등록 완료율 | 등록 완료 수 | 등록 시작 수 | 전체/일간 |
| 만료율 | 만료 완료 쿠폰 수 | 등록 완료 쿠폰 수 | 전체/주간 |
| 알림 오픈율 | 알림 오픈 수 | 알림 발송 수 | 전체/일간 |
| 상세 조회율 | 상세 조회 쿠폰 수 | 등록 완료 쿠폰 수 | 전체/주간 |

기간 정의 원칙:
- MVP에서는 전체 기간 누적 지표를 우선 사용한다.
- 필요 시 일 단위/주 단위로 추가 분해한다.
- 쿠폰 상태 기반 지표는 쿠폰 기준 집계, 알림 지표는 이벤트 기준 집계를 원칙으로 한다.

## 7. 이번 MVP에서 측정 가능한 범위와 측정 불가능한 범위

### 측정 가능한 범위
- 등록 시작 수
- 등록 완료 수
- 상세 조회 수
- 사용 완료 수
- 만료 수
- 알림 발송 수
- 알림 오픈 수
- 쿠폰 기준 전환율
- 기본 퍼널 전환
- 캠페인/브랜드 유사 단위별 비교

### 측정 불가능하거나 제한적인 범위
- 실제 매장 결제 여부
- 알림을 봤지만 클릭하지 않은 경우의 정확한 반응
- 로그인 기반 사용자 단위 크로스디바이스 추적
- 서버 기준 DAU/WAU/MAU
- 푸시 전달 실패 여부
- 통계적으로 유의미한 인과 해석

## 운영 메모
- SQLite, SQL, Python 노트북, 대시보드에서 KPI 이름과 계산식을 동일하게 유지한다.
- KPI 계산 시 가능하면 이벤트 기준과 쿠폰 기준을 혼동하지 않도록 구분한다.
- 이번 MVP에서는 "정확한 실무 운영 지표 체계"보다 "설명 가능한 소규모 지표 체계"를 우선한다.
