# 앱 개인정보 라벨 입력 가이드

이 문서는 App Store Connect의 **앱 개인정보 처리 방침(App Privacy / Privacy Nutrition Label)** 입력 시 참고하기 위한 가이드입니다.

쿠왕은 로그인과 쿠왕 자체 서버 저장이 없으며 쿠폰/멤버십/이미지 데이터는 기기 안에 로컬로 저장됩니다. 다만 앱에는 Google AdMob 배너 광고가 포함되므로, App Store Connect의 개인정보 라벨에는 Google Mobile Ads SDK가 처리할 수 있는 데이터 항목을 함께 반영해야 합니다.

## 1. 입력 전 기준 정리

| 기준 | 쿠왕 현재 정책 |
|---|---|
| 계정/로그인 | 없음 |
| 서버 저장 | 없음 |
| 쿠왕 자체 외부 전송 | 없음 |
| 광고 | Google AdMob 배너 광고 포함 |
| 사용자 추적 | AdMob 개인 맞춤 광고/ATT 설정에 따라 최종 확인 필요 |
| 쿠폰/멤버십 데이터 | 기기 내 로컬 저장 |
| 이미지 데이터 | 사용자가 선택한 이미지를 기기 내 앱 저장공간에 저장 |
| 알림 | 서버 푸시가 아닌 로컬 알림 |

## 2. 데이터 수집 여부

### 선택값

**데이터 수집함**

### 이유

쿠왕이 저장하는 쿠폰, 멤버십, 이미지 정보는 기기 내에만 저장되지만, 앱에 Google AdMob SDK가 포함되어 광고 제공, 광고 측정, 보안 및 부정 사용 방지 목적으로 일부 데이터가 처리될 수 있습니다.

## 3. 중요 입력 주의사항

App Store Connect에서 **데이터 수집함**을 선택한 뒤, AdMob SDK가 처리할 수 있는 데이터 유형을 세부 항목에 반영합니다.

아래 입력값은 Google Mobile Ads SDK를 사용하는 현재 구조 기준의 권장 초안입니다. App Store Connect 제출 직전에는 Google Mobile Ads SDK의 최신 데이터 공시 문서와 실제 광고 설정(개인 맞춤 광고, ATT, UMP 적용 여부)을 다시 확인합니다.

권장 입력은 다음과 같습니다.

| 상황 | 권장 선택 |
|---|---|
| 쿠폰/멤버십/이미지는 기기 내 저장이고 AdMob 배너 광고가 포함된 현재 구조 | 데이터 수집함 |
| 향후 이미지가 서버, 클라우드, 분석 도구, OCR 서버 등 외부로 전송되는 구조로 바뀌는 경우 | 사용자 콘텐츠 > 사진 또는 동영상 수집함 |

현재 쿠왕 v1.0 기준으로는 **데이터 수집함**을 선택하는 것이 AdMob 포함 구조와 일치합니다.

## 4. 데이터 유형별 입력 가이드

아래 표는 App Store Connect 화면 순서에 맞춘 확인용 체크리스트입니다.

| 데이터 유형 | 선택값 | 이유 |
|---|---|---|
| 연락처 정보 | 수집 안 함 | 이름, 이메일, 전화번호, 주소를 수집하지 않음 |
| 건강 및 피트니스 | 수집 안 함 | 건강/운동 관련 기능 없음 |
| 재무 정보 | 수집 안 함 | 결제, 신용카드, 금융 정보 기능 없음 |
| 위치 | 대략적 위치 수집 가능 | AdMob SDK가 IP 주소로 대략적 위치를 추정할 수 있음 |
| 민감한 정보 | 수집 안 함 | 민감정보 입력 또는 처리 기능 없음 |
| 연락처 | 수집 안 함 | 기기 연락처 접근 없음 |
| 사용자 콘텐츠 | 수집 안 함 | 현재 구조에서는 사진/쿠폰 이미지를 외부로 전송하지 않고 기기 내에만 저장 |
| 식별자 | 수집 가능 | AdMob SDK가 광고 ID, 기기 ID 등 식별자를 광고 제공/측정 목적으로 처리할 수 있음 |
| 구매 | 수집 안 함 | 인앱 구매 또는 결제 기능 없음 |
| 사용 데이터 | 수집 가능 | AdMob SDK가 앱 실행, 광고 노출/탭 등 광고 관련 상호작용을 처리할 수 있음 |
| 진단 | 수집 가능 | AdMob SDK가 SDK 성능, 오류, 충돌 관련 정보를 처리할 수 있음 |
| 기타 데이터 | 수집 안 함 | 위 항목 외 수집 데이터 없음 |

## 5. 사용자 콘텐츠 항목 보조 기준

요청서 기준의 "사진 또는 동영상" 항목을 App Store Connect에서 별도로 입력해야 하는 상황이 생기면 아래처럼 판단합니다.

### 현재 v1.0 권장 선택

| 항목 | 선택값 | 이유 |
|---|---|---|
| 사용자 콘텐츠 > 사진 또는 동영상 | 수집 안 함 | 이미지는 사용자의 기기 내 앱 저장공간에만 저장되고 개발자 또는 제3자에게 전송되지 않음 |

### 향후 서버 전송 기능이 생기는 경우의 대체 선택

| 항목 | 선택값 |
|---|---|
| 사용자 콘텐츠 > 사진 또는 동영상 | 수집함 |
| 목적 | 앱 기능 |
| 사용자 식별에 연결 | 아니오 |
| 추적 목적 | 아니오 |

이 대체 선택은 현재 쿠왕 v1.0에는 적용하지 않습니다. 향후 이미지가 외부 서버로 전송되는 구조가 생길 때만 사용합니다.

## 6. 추적 여부

### 선택값

**이 앱은 사용자 또는 기기를 추적하지 않습니다**

### 이유

쿠왕 자체는 사용자 데이터를 제3자 데이터와 결합하거나 별도 추적 시스템을 운영하지 않습니다. 단, AdMob 개인 맞춤 광고 또는 IDFA 기반 광고 측정을 사용하면 Apple 기준의 "추적"에 해당할 수 있습니다.

### 제출 전 확인

| 설정 | 권장 판단 |
|---|---|
| ATT 요청/개인 맞춤 광고를 사용하지 않는 경우 | 추적 안 함으로 제출 가능성 높음 |
| ATT 요청, IDFA 사용, 개인 맞춤 광고, 타사 데이터 결합이 있는 경우 | 추적함으로 제출 필요 |

현재 코드와 AdMob 콘솔 설정을 기준으로 최종 선택합니다.

## 7. 개인정보 처리방침 URL

### 선택값

개인정보처리방침을 배포한 공개 URL 입력

### 입력 전 준비

| 항목 | 상태 |
|---|---|
| 한국어 개인정보처리방침 | `docs/legal/privacy_policy_ko.md` 기준으로 웹 게시 필요 |
| 영어 Privacy Policy | `docs/legal/privacy_policy_en.md` 기준으로 웹 게시 필요 |
| 한국어 HTML 예상 URL | `https://hdyeon67.github.io/CouWang/legal/privacy_policy_ko.html` |
| 영어 HTML 예상 URL | `https://hdyeon67.github.io/CouWang/legal/privacy_policy_en.html` |
| 대표 개인정보처리방침 URL | `https://hdyeon67.github.io/CouWang/legal/privacy_policy_ko.html` |

## 8. 최종 제출 전 체크리스트

| 체크 항목 | 확인 |
|---|---|
| 로그인/회원가입이 없는가 | 예 |
| 서버 또는 Firebase로 사용자 데이터가 전송되지 않는가 | 예 |
| 광고 SDK가 포함되어 있는가 | 예. Google AdMob |
| 사진/이미지가 외부 서버로 업로드되지 않는가 | 예 |
| 알림이 로컬 알림으로만 동작하는가 | 예 |
| 개인정보처리방침 URL이 공개 접근 가능한가 | 제출 전 확인 필요 |
| App Store Connect에서 "데이터 수집함" 선택이 현재 구현과 일치하는가 | 예 |
