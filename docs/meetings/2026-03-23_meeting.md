## 회의 목적
- 앱 스토리보드와 실제 MVP 화면 구성을 다시 점검한다.
- 7일 안에 완성 가능한 범위로 기능을 줄이고, 구현 우선순위를 확정한다.
- 홈 화면, 쿠폰 관리 흐름, 멤버십/알림 구조를 실제 사용 흐름 기준으로 재정리한다.

## 프로젝트 방향 재정리
- 이번 MVP는 복잡한 AI 추천 기능보다 실제로 동작하는 쿠폰/멤버십 관리 흐름을 우선한다.
- 서비스의 핵심은 쿠폰과 멤버십을 잊지 않게 하고, 만료 전에 실제 사용으로 연결하는 것이다.
- 화면 수를 늘리기보다 홈, 등록, 상세, 알림 흐름의 완성도를 높이는 방향으로 정리한다.

## 주요 논의 내용

### 1. MVP 범위 조정
- API 연동이나 구현 비용이 큰 AI 기능은 MVP에서 제외한다.
- 스마트 제안, 고도화된 추천, 복잡한 자동화 기능은 후순위로 둔다.
- 이번 단계에서는 실제로 실행 가능한 앱 구조와 사용자 흐름을 우선한다.

### 2. 홈 화면과 쿠폰 리스트 통합
- 홈 화면과 쿠폰 리스트 화면을 분리하지 않고 하나의 화면으로 통합한다.
- 사용자는 앱에 들어오자마자 현재 보유 쿠폰과 만료 임박 상태를 바로 확인할 수 있어야 한다.
- 홈 상단에는 마스코트와 절약 리포트 성격의 메시지를 배치해 서비스 정체성을 전달한다.
- 하단 플로팅 버튼으로 등록 진입을 일관되게 제공한다.

### 3. 쿠폰 리스트 구성 방식
- 쿠폰 리스트는 기본적으로 만료 임박순 정렬을 사용한다.
- 임박 쿠폰은 카드 강조, D-day 배지 등으로 시각적 경각심을 준다.
- 검색과 정렬은 확장 가능하게 두되, MVP 기준에서는 단순하고 설명 가능한 수준을 유지한다.

### 4. 쿠폰 등록 구조
- 수동 입력을 기본 등록 방식으로 유지한다.
- 이미지 업로드 기반 자동 입력과 바코드/QR 인식은 보조 흐름으로 연결한다.
- 데이터 활용과 실제 사용 흐름을 고려할 때 이미지 첨부의 중요성이 높다는 점을 다시 확인했다.
- 다만 필수값을 과도하게 늘리면 이탈이 생길 수 있으므로 실제 저장 구조에서는 균형이 필요하다.

### 5. 쿠폰 상세 및 관리
- 상세 화면에서는 이미지 확대, 바코드 확인, D-day 확인이 직관적으로 가능해야 한다.
- 수정/삭제는 메인 CTA보다 보조 관리 액션으로 두는 방향이 적절하다고 봤다.
- 유효기간은 복잡한 그래프보다 텍스트와 D-day 중심으로 명확히 보여주는 것이 좋다고 정리했다.

### 6. 멤버십과 알림 구조
- 멤버십은 별도 리스트로 접근하되, 쿠폰과 함께 관리되는 흐름이 보이게 해야 한다.
- 알림은 단순 ON/OFF가 아니라 30일, 7일, 3일, 1일, 당일처럼 세부 주기를 조정할 수 있는 구조가 필요하다.
- 설정 탭의 존재 자체보다 실제 사용자가 기능을 어디에서 찾는지가 더 중요하다는 의견이 있었다.

### 7. 브랜드 컬러와 캐릭터 방향
- 브랜드 캐릭터는 사실적인 강아지보다 단순하고 친근한 카툰 스타일로 정리하기로 했다.
- 키 컬러는 `#64CAFA`로 확정하고, 배경은 미세하게 푸른빛이 도는 밝은 회색 톤을 사용하기로 했다.
- 캐릭터 가시성을 높이기 위한 포인트 컬러로 노란색 계열을 보조색으로 활용하는 방향을 논의했다.

### 8. 오후 회의 기준 홈 구조와 탭 구조 정리
- 메인 홈 화면에는 로고, 알림 버튼, 리포트 배너, 쿠폰 리스트가 들어가는 구조를 유지한다.
- 쿠폰 리스트는 만료순 또는 이름순 정렬이 가능하고, 검색은 서버 연동 없이 로컬 검색으로 구현한다.
- 하단 탭 바는 `홈(쿠폰 리스트)`, `멤버십 리스트`, `설정` 화면에서만 공통으로 노출하는 방향으로 정리했다.

### 9. 데이터 및 분석 참고사항
- 분석용 데이터는 실제 서비스 로그가 아니라 합성(synthetic) 데이터라는 점을 명확히 인지하기로 했다.
- FAQ 기준으로 퍼널, 쿠폰, 알림, 멤버십, A/B 테스트는 각각 다른 테이블 조합으로 봐야 한다.
- 자동 입력 관련 이벤트는 `coupon_auto_scan`, `coupon_auto_success`, `coupon_auto_fail` 흐름으로 해석한다.
- 푸시 퍼널은 `push_received → push_open → push_click → coupon_detail_view → coupon_use_complete` 순서로 보는 기준을 공유했다.

### 10. 용어와 정보 구조 정리
- 브랜드, 사용처, 쿠폰명, 카테고리 등 용어를 화면과 데이터 구조에서 통일해야 한다.
- 기능을 추가하는 것보다 이미 있는 화면이 더 직관적으로 보이도록 정리하는 것이 우선이라는 점에 공감대가 있었다.

## 핵심 결정 사항
- 복잡한 AI 기능은 MVP 범위에서 제외한다.
- 홈과 쿠폰 리스트는 하나의 화면으로 통합한다.
- 쿠폰 리스트 기본 정렬은 만료 임박순으로 둔다.
- 쿠폰 상세에서는 D-day와 유효기간을 명확히 보여준다.
- 멤버십은 별도 리스트 구조로 유지한다.
- 알림은 세부 주기 조정이 가능한 구조를 유지한다.
- 브랜드 키 컬러는 `#64CAFA`를 기준으로 사용한다.
- 하단 탭은 홈/멤버십/설정 공통 구조를 기본 후보로 유지한다.
- 버튼, 필드, 메뉴 명칭은 문서와 화면에서 통일한다.

## 2026-03-23 작업 반영 내용

- 하단 탭 구조를 쿠폰/알림 중심으로 단순화하고, 이후 공통 탭 구조를 재정리하는 방향으로 수정했다.
- 쿠폰 등록 화면에 자동 입력 흐름을 연결했다.
  - 갤러리 이미지 기반 자동 입력
  - 바코드/QR 코드 인식
  - OCR 기반 텍스트 추출
- OCR에서 날짜 범위 표기 시 마지막 날짜를 만료일로 인식하도록 보정했다.
- 멤버십 리스트, 등록, 상세 화면 초안을 구현했다.
- 알림 리스트와 알림 설정 화면 흐름을 연결했다.
- 홈 A/B 구조를 실험한 뒤 최종 홈 구조 후보로 리팩토링하는 방향이 정리되었다.
