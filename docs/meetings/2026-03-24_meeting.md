## 오전 회의 핵심
- 캐릭터 3종과 앱 UI 방향을 확정했다.
- 쿠폰 사용 시 발바닥 도장 연출 등 마스코트 활용 포인트를 정리했다.
- 다크 모드에서도 캐릭터가 잘 보이도록 아웃라인을 보정하기로 했다.

- 로그인, 온보딩 같은 이벤트는 현재 서비스 설계와 맞지 않아 분석에서 제외하기로 했다.
- 멤버십 미사용 비율은 오류가 아니라 활성화 과제로 해석하기로 했다.

## 데이터 해석 및 전처리 결정
- 앱 설치 후 홈 진입 전 대규모 이탈은 사용자 행동뿐 아니라 시스템 오류 가능성도 함께 고려하기로 했다.
- 데이터는 새로 생성하지 않고, 불필요한 이벤트를 드롭하는 방식으로 정리하기로 했다.
- `coupon.csv`의 금액 정보는 홈 절약 리포트에 필요하다고 판단했다.
- 가격이 없는 쿠폰을 보완하기 위해 `coupon_amount.csv` 같은 보조 테이블이 필요하다는 점을 정리했다.
- `store`와 `brand`는 중복 가능성이 있어도 원본 보존 차원에서 유지하기로 했다.
- `membership.csv`의 `has_image`는 설계상 필수값으로 보고 해석하기로 했다.
- `event_logs`는 서비스 구조와 맞지 않는 이벤트를 제거한 전처리 버전을 기준으로 보기로 했다.

## 오후 회의 핵심
- 분석 파트와 기획/솔루션 파트를 나누어 병렬로 진행하기로 했다.
- 핵심 문제를 세 갈래로 정리했다.
  - 가입 후 쿠폰 등록 단계로 넘어가지 못하는 초기 활성화 문제
  - 쿠폰 등록 후 실제 사용 단계까지 이어지지 않는 망각/전환 문제
  - 회원가입 절차 내부의 심리적·기술적 허들 문제
- 푸시 알림은 전체 오픈율만 보는 것이 아니라 발송 시점별 반응 차이를 나눠 해석하기로 했다.
- 쿠폰 삭제와 사용 완료 행동을 비교해, 완료 버튼 클릭 유도 전략으로 연결하기로 했다.

## 세부 분석 포인트
- `회원가입 완료 → 쿠폰 등록 시작` 전환이 주요 이탈 구간이라는 점을 재확인했다.
- `쿠폰 등록 완료 → 실제 사용 완료` 전환도 별도 문제 구간으로 보기로 했다.
- 자동 스캔 성공/실패 비율 차이가 존재해 OCR/자동 입력 UX 품질을 점검할 필요가 있다고 봤다.
- 멤버십과 푸시, 채널, 쿠폰 사용 완료까지 각각 담당자를 나누어 다시 분석하기로 했다.
