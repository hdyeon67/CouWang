## 오전 회의 핵심 정리
- 캐릭터와 앱 UI 방향을 확정했다.
- 쿠폰 사용 시 발바닥 모양 도장 연출 같은 캐릭터 활용 아이디어를 정리했다.
- 다크 모드에서도 캐릭터가 잘 보이도록 아웃라인 보정 방향을 잡았다.
- 제공 데이터셋과 현재 서비스 설계 간의 차이를 다시 점검했다.
- 로그인, 온보딩 같은 이벤트는 현재 앱 설계와 맞지 않으므로 분석 대상에서 제외하는 방향을 잡았다.
- 멤버십 미사용 비율은 데이터 오류보다는 활성화 과제로 보고 해석하기로 했다.

## 데이터 해석 및 전처리 결정
- 앱 설치 후 홈 진입 전 대규모 이탈 구간은 사용자 행동보다 시스템적 오류 가능성까지 포함해 해석하기로 했다.
- 데이터를 새로 생성하거나 크게 수정하지 않고, 현재 데이터 안에서 불필요 이벤트를 드롭하는 방향으로 진행하기로 했다.
- `coupon.csv`의 금액 정보는 홈 상단 절약 리포트 계산에 필요하다고 판단했다.
- 금액이 없는 쿠폰을 보완하기 위해 `coupon_amount.csv` 형태의 보조 데이터가 필요하다는 점을 정리했다.
- `store`와 `brand` 컬럼은 중복 가능성이 있어도 원본 데이터 보존 차원에서 유지하기로 했다.
- `membership.csv`의 `has_image`는 설계상 필수값으로 보고 해석하기로 했다.
- `event_logs`는 현재 서비스 구조와 맞지 않는 이벤트를 제외한 전처리 버전을 기준으로 보기로 했다.

## 오후 회의 핵심 정리
- 분석 파트와 기획/솔루션 파트로 나누어 병렬로 진행하기로 했다.
- 주요 문제 정의를 세 갈래로 정리했다.
  - 가입 완료 후 쿠폰 등록 단계로 넘어가지 못하는 초기 활성화 문제
  - 쿠폰 등록 후 실제 사용 단계까지 이어지지 않는 망각/전환 문제
  - 회원가입 절차 내부의 심리적·기술적 허들 문제
- 푸시 알림은 전체 오픈율만 보기보다 발송 시점별 반응 차이를 나눠서 봐야 한다는 점을 확인했다.
- 쿠폰 삭제와 사용 완료의 차이를 로그 기준으로 비교해, 완료 버튼 클릭 유도 전략으로 연결하기로 했다.

## 3월 24일 앱 작업 반영 사항
- 홈/멤버십/설정 공통 하단 탭 구조를 유지하면서 화면 톤을 정리했다.
- 멤버십 리스트 화면을 홈과 같은 디자인 시스템으로 리디자인했다.
- 쿠폰 등록 흐름을 단순화했다.
  - 홈 FAB 클릭 시 바로 쿠폰 등록 화면으로 이동
  - 자동 입력/수동 입력 선택 바텀시트 제거
- 쿠폰 등록 화면에서 이미지 업로드 후 OCR 버튼이 활성화되는 흐름으로 정리했다.
- 웹에서는 이미지 미리보기가 깨지지 않도록 이미지 표시 방식을 정리했다.
- 인증 흐름을 `AuthEntryScreen` 단일 구조로 정리했다.
  - 로딩
  - 로그인 버튼 노출
  - 로그인 완료 시 홈 이동
- Google 로그인 연동을 위한 Firebase 연결 작업을 진행했다.
  - Android/iOS 식별자를 `com.fineboll.couwangApp`으로 맞춤
  - `google-services.json`, `GoogleService-Info.plist` 추가
  - `firebase_core`, `firebase_auth`, `google_sign_in` 연결
- Android에서는 Google 로그인 성공까지 확인했다.
- iOS에서는 빌드/설치 및 실행 attach 단계에서 흰 화면 이슈가 있어 계속 점검 중이다.

## 현재 확인된 이슈
- iOS는 빌드 자체보다 설치 후 실행 및 attach 단계가 불안정하다.
- `Attaching to Runner` 이후 흰 화면으로 멈추는 현상이 있어, Firebase 초기화 위치와 시작 흐름을 조정했다.
- Android는 Firebase와 Google 로그인 설정이 반영된 상태로 재검증을 이어갈 수 있다.

## 결정 사항
- 데이터는 원본 유지 원칙을 따른다.
- 현재 서비스 설계와 맞지 않는 온보딩/회원가입 이벤트는 분석에서 제외한다.
- 홈 리포트는 금액 기반 절약 체감 메시지를 유지한다.
- 인증 화면은 별도 AuthGate와 LoginScreen을 분리하지 않고 단일 진입 화면으로 유지한다.
- Google 로그인부터 우선 구현하고, Kakao는 이후 단계로 둔다.
