# 작업 노트와 결정사항

날짜별 작업 정리와 프로젝트 의사결정을 관리합니다.

| 파일 | 역할 |
|---|---|
| `decision_log.md` | 제품/기술/출시 관련 최종 결정사항 로그 |
| `YYYY-MM-DD_*.md` | 날짜별 작업 내용, 변경 이유, 검증 결과 |

## 관리 기준

- 하루 동안 여러 성격의 작업이 섞이면 날짜별 노트에는 전체 흐름을 남기고, 세부 산출물은 각 담당 폴더에 둡니다.
- 최종적으로 유지해야 할 정책은 `decision_log.md`에 짧게 남깁니다.
- 오래된 판단이 바뀌면 기존 내용을 지우기보다 새 날짜에 변경 결정을 추가합니다.
