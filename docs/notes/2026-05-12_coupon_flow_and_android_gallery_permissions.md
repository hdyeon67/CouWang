# 2026-05-12 작업 정리

## 오늘 한 일

- 쿠폰 등록 화면에서 이미지 추출 뒤 필수값이 일부 비어 있을 때, 누락된 항목명을 스낵바로 바로 안내하도록 보완했다.
- 쿠폰 저장 뒤 알림 재예약이나 분석 로그 처리에서 예외가 나도 전체 저장 실패처럼 보이지 않도록 후처리 예외를 분리했다.
- 쿠폰 상세 화면에서 삭제 후 화면 스택에 따라 멈추지 않도록 홈 화면으로 안정적으로 복귀하는 흐름으로 정리했다.
- 갤러리 자동 감지에서 이미 저장된 쿠폰을 다시 등록 후보로 띄우지 않도록, 바코드 우선과 제목+유효기간 보조 기준으로 중복 감지를 추가했다.
- 토스뱅크 계열 쿠폰에서 `유효기간 2026.05.14`처럼 보이는 날짜가 OCR 결과에서 공백, 점, 붙은 숫자 형태로 흔들려도 인식할 수 있게 날짜 파서를 보강했다.
- Android 갤러리 접근 기능에 실제 필요한 권한을 다시 확인하고, `READ_MEDIA_IMAGES`와 `READ_EXTERNAL_STORAGE(maxSdkVersion=32)`만 유지하도록 정리했다.

## 변경 이유

- OCR 결과가 부분 성공했을 때 사용자가 무엇을 직접 입력해야 하는지 바로 알기 어려웠다.
- 저장은 성공했는데 후처리 예외 때문에 `couponSaveFailed`처럼 보이는 흐름이 사용자 경험을 해치고 있었다.
- 갤러리 자동 감지의 중복 제안과 일부 쿠폰 이미지의 날짜 인식 누락이 반복적으로 확인됐다.
- Android 권한은 최소 범위로 유지하되, 실제 갤러리 자동 감지와 이미지 선택 흐름에 필요한 권한은 다시 명확히 남길 필요가 있었다.

## 검증

- `flutter analyze` 통과
- AndroidManifest git history 확인
  - `READ_MEDIA_IMAGES`, `READ_EXTERNAL_STORAGE` 최초 추가 커밋: `40ec0cd`
  - `READ_MEDIA_VIDEO`는 manifest 이력에 없음

## 메모

- Android 갤러리 관련 기능은 현재 구조상 이미지 읽기 권한이 필요하다.
- 현재 유지 권한은 이미지 접근에 필요한 최소 범위로 한정하고, 동영상 읽기 권한은 추가하지 않는다.
