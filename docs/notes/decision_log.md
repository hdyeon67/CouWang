# 쿠왕 결정사항 로그

## 2026-03-23

### 제품 방향
- 이번 MVP는 복잡한 AI 기능보다 실제로 동작하는 쿠폰/멤버십 관리 흐름에 집중한다.
- 홈 화면과 쿠폰 리스트 화면은 하나의 화면으로 통합한다.
- 사용자는 앱에 들어오자마자 현재 보유 쿠폰과 만료 임박 상태를 바로 확인할 수 있어야 한다.

### 홈 화면
- 홈은 `쿠폰 사용 유도형 대시보드` 성격으로 정리한다.
- 상단에는 마스코트와 절약 리포트 메시지를 배치한다.
- 쿠폰 리스트 기본 정렬은 만료 임박순으로 둔다.
- 임박 쿠폰은 D-day 배지와 카드 강조로 눈에 띄게 표현한다.
- 브랜드 키 컬러는 `#64CAFA`를 기준으로 사용한다.

### 쿠폰 등록
- 수동 입력을 기본 등록 방식으로 유지한다.
- 이미지 업로드 기반 자동 입력은 보조 흐름으로 제공한다.
- 바코드/QR 코드 인식과 OCR은 등록 보조 기능으로 연결한다.
- 등록 필수값은 너무 무겁지 않게 유지하되, 이미지 첨부의 중요성은 높게 본다.

### 쿠폰 상세
- 상세 화면에서는 D-day와 유효기간을 명확히 보여준다.
- 이미지 확대와 바코드 확인이 가능해야 한다.
- 수정/삭제는 메인 CTA보다 보조 관리 액션으로 둔다.

### 멤버십/알림
- 멤버십은 별도 리스트 구조로 유지한다.
- 알림은 단순 ON/OFF가 아니라 세부 주기 조정이 가능한 구조를 지향한다.
- 기능 위치는 설정 탭 존재 자체보다 사용자가 쉽게 찾을 수 있는 흐름을 우선한다.

### 구현 기준
- 버튼, 필드, 메뉴 명칭은 문서와 화면에서 통일한다.
- 기능을 늘리기보다 기존 화면을 더 직관적으로 다듬는 것을 우선한다.
- 분석 데이터는 실제 로그가 아닌 합성 데이터라는 점을 문서와 발표에서 명확히 구분한다.
