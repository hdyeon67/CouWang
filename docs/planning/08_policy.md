# 정책 문서

## 1. 데이터 정책

| 항목 | 정책 |
|---|---|
| 저장 방식 | 현재 MVP는 서버 없이 로컬 SQLite(`kuwang_local.db`)에 저장한다. |
| 저장 대상 | 쿠폰, 멤버십, 이미지 메타데이터, 알림 설정, 알림 로그 |
| 이미지 저장 | 이미지 파일은 앱 문서 디렉터리의 `images/{ownerType}/...` 하위에 저장하고 DB에는 상대 경로와 메타데이터를 저장한다. |
| 삭제 정책 | 쿠폰/멤버십 삭제 시 연결된 이미지 파일과 `image_assets` 메타데이터를 삭제한다. 삭제된 쿠폰/멤버십은 앱 내에서 복구할 수 없다. |
| 알림 로그 삭제 | 알림 로그는 `is_deleted` 상태로 관리하며, 화면에서는 삭제된 알림을 표시하지 않는다. |
| 백업 정책 | 현재 자동 백업은 제공하지 않는다. 향후 로그인/서버/이메일 백업을 검토한다. |
| 개인정보 | 현재 계정, 이메일, 이름 같은 사용자 계정 개인정보는 수집하지 않는다. 사용자가 입력한 쿠폰/멤버십 정보는 기기 로컬에 저장된다. |

## 2. 알림 정책

| 항목 | 정책 |
|---|---|
| 알림 방식 | `flutter_local_notifications` 기반 로컬 알림을 사용한다. 실제 푸시 서버/Firebase는 사용하지 않는다. |
| 알림 발송 시각 | 현재 구현 기준 정오 12시로 예약한다. |
| 타임존 | `Asia/Seoul` 기준으로 예약한다. |
| 알림 대상 조건 | 마스터 알림이 켜져 있고, 쿠폰이 미사용이며, 만료 상태가 아닌 경우만 예약한다. |
| 알림 타입 | D-30, D-7, D-3, D-1, D-DAY, 만료 다음날 알림을 지원한다. |
| 마스터 토글 OFF | 전체 예약 알림을 취소한다. |
| 세부 토글 OFF | 해당 타입의 알림은 예약하지 않는다. |
| 쿠폰 삭제 | 해당 쿠폰의 예약 알림과 관련 로그를 정리한다. |
| 사용 완료 처리 | 해당 쿠폰의 예약 알림을 취소한다. |
| 사용 미완료 처리 | 쿠폰이 미만료 상태이면 알림을 다시 예약한다. |
| 앱 재시작 | 앱 시작 또는 설정 변경 시 저장된 쿠폰/설정 기준으로 알림 재스케줄을 수행한다. |
| Android 13 이상 | 시스템 알림 권한 요청을 사용한다. |
| Android 12 이하 | 시스템 런타임 알림 권한 팝업이 없으므로 앱 내부 안내 다이얼로그로 동의받고 내부 알림 설정을 켠다. |
| 웹 | Chrome에서는 로컬 알림 플러그인 호출을 건너뛰어 앱 실행을 우선한다. |

## 3. D-DAY 계산 정책

| 항목 | 정책 |
|---|---|
| 계산 기준 | 현재 날짜의 자정과 만료일 자정의 차이를 일수로 계산한다. |
| D-DAY | 차이가 0이면 만료 당일로 보고 `D-DAY`로 표시한다. |
| 만료 | 차이가 음수이면 만료 상태로 처리한다. |
| 임박 | 차이가 1-3일이면 urgent 상태로 분류한다. |
| 사용완료 표시 | 사용 완료 쿠폰은 D-DAY 대신 `사용완료` 라벨을 우선 표시한다. |
| 만료 표시 | 만료 쿠폰은 D-DAY 대신 `만료` 또는 상세 화면의 `기간만료` 라벨을 표시한다. |
| 색상 기준 | D-DAY에 가까울수록 하늘색 계열을 진하게 표시하고, 사용완료/만료는 하늘색 계열이 아닌 회색/상태색 계열로 분리한다. |

## 4. 쿠폰 상태 정책

| 항목 | 정책 |
|---|---|
| 상태 종류 | 사용가능, 임박, 사용완료, 만료를 내부 상태로 관리하고 화면에서는 사용가능/사용완료/만료 중심으로 필터링한다. |
| 사용가능 | 미사용이며 만료일이 오늘 또는 미래인 쿠폰 |
| 임박 | 미사용이며 D-3 이하인 쿠폰 |
| 사용완료 | 사용자가 사용 완료 처리한 쿠폰 |
| 만료 | 만료일이 오늘보다 이전인 쿠폰 |
| 사용 완료 되돌리기 | 현재 구현에서는 사용 완료 쿠폰을 확인 다이얼로그 후 사용 미완료로 되돌릴 수 있다. |
| 만료 기준 | 만료일 다음날 00:00부터 만료 상태로 본다. |

## 5. 이미지 처리 정책

| 항목 | 정책 |
|---|---|
| 이미지 소스 | 갤러리 이미지만 사용한다. 실시간 카메라 스캔은 제공하지 않는다. |
| 지원 포맷 | UI 안내 기준 PNG, JPG를 지원한다. 저장 서비스는 확장자 기반으로 JPEG/PNG/WebP mime type을 추정한다. |
| 최대 용량 | UI 안내는 최대 10MB이다. 단, 현재 코드에서 저장 전 10MB hard validation은 별도 보강 필요 항목으로 남긴다. |
| 저장 위치 | 앱 문서 디렉터리 `images/coupons` 또는 `images/memberships` 하위에 저장한다. |
| OCR 처리 | 이미지가 선택된 후 정보 추출 버튼으로 텍스트/바코드/QR 인식을 수행한다. |
| 이미지 미등록 | 쿠폰/멤버십 상세에서는 기본 배경색, 아이콘 또는 텍스트 대체 UI를 표시한다. |
| 삭제 | 쿠폰/멤버십 삭제 시 연결 이미지 파일도 삭제한다. |

## 6. UI/UX 정책

| 항목 | 정책 |
|---|---|
| 저장 버튼 | 필수 입력값이 비어 있으면 비활성화한다. |
| 삭제 | 쿠폰, 멤버십, 알림 삭제는 확인 다이얼로그를 거친다. |
| 상태 변경 | 사용 완료/사용 미완료 처리는 확인 다이얼로그를 거친다. |
| 스낵바 | 주요 완료 안내는 2초 내외의 floating SnackBar로 표시한다. |
| 하단 탭 | 멤버십/홈/설정 하단 탭을 주요 루트 화면에 고정 제공한다. |
| Empty 상태 | 빈 목록에는 `assets/icon/4.png` 기반 공통 마스코트 empty UI를 사용한다. |
| 검색 필드 | TextField가 라운드 border를 침범하지 않도록 내부 높이와 좌우 inset을 조정한다. |
| 뒤로가기 | 커스텀 뒤로가기 아이콘은 좌측 여백을 줄여 화면 가장자리와 자연스럽게 맞춘다. |

## 7. 심사/출시 정책

| 서버 | 현재 미도입. 서버 도입 시 데이터 전송, 암호화, 백업 정책을 재작성한다. |
