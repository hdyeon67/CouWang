# 기획 문서

쿠왕의 제품 요구사항, 화면 흐름, 기능 정의, 정책을 관리합니다.

| 파일 | 역할 |
|---|---|
| `01_requirements.md` | MVP 요구사항과 범위 |
| `02_screen_flow.md` | 사용자 화면 흐름 |
| `03_screen_design.md` | 화면별 UI 설계 |
| `04_functional_spec.md` | 기능 명세 |
| `05_storyboard.md` | 사용자 시나리오 |
| `06_feature_definition.md` | 기능 정의 |
| `07_information_architecture.md` | 정보 구조 |
| `08_policy.md` | 알림, 권한, 데이터 등 정책 |
| `export/` | 기획 문서에서 추출한 CSV/시트 자료 |

## 관리 기준

- 큰 방향과 MVP 범위는 `01_requirements.md`를 기준으로 봅니다.
- 구현 중 변경된 정책은 `08_policy.md`와 `notes/decision_log.md`에 함께 반영합니다.
- 화면 문구나 스토어 문구는 `store/` 문서와 충돌하지 않게 유지합니다.
