# 기능 명세서

## 1. 쿠폰 CRUD

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-001 |
| 기능명 | 쿠폰 등록/조회/수정/삭제 |
| 설명 | 사용자가 쿠폰 정보를 로컬 SQLite에 저장하고 리스트/상세에서 관리한다. |
| 입력값 | 쿠폰명, 브랜드, 카테고리, 코드값, 코드타입, 유효기간, 메모, 이미지 |
| 처리 로직 | `CouponRepository.saveDraft`가 쿠폰 ID를 생성 또는 유지하고, `coupons` 테이블에 upsert한다. 이미지가 있으면 `LocalImageStorageService`로 파일 저장 후 `image_assets`에 메타데이터를 저장한다. |
| 출력/결과 | 저장된 쿠폰이 Repository 캐시와 UI에 반영된다. |
| 예외 처리 | 필수값 미입력 시 저장 버튼 비활성화. 삭제 시 확인 다이얼로그 표시. 삭제 시 이미지 파일과 `image_assets` 메타데이터 삭제. |

## 2. 쿠폰 사용 완료 처리

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-002 |
| 기능명 | 쿠폰 사용 완료/사용 미완료 처리 |
| 설명 | 쿠폰 상세에서 사용 완료 상태를 변경한다. |
| 입력값 | couponId, 사용 완료 여부 |
| 처리 로직 | 사용 완료는 `CouponRepository.markUsed`, 사용 미완료는 `CouponRepository.markUnused`를 호출한다. 사용 완료 시 예약 알림을 취소하고, 사용 미완료 시 미만료 쿠폰이면 알림을 재등록한다. |
| 출력/결과 | 상세 CTA 문구와 리스트 상태가 변경된다. |
| 예외 처리 | 확인 다이얼로그에서 취소하면 상태를 변경하지 않는다. 만료 쿠폰은 상태 계산 결과에 따라 만료로 복원될 수 있다. |

## 3. 쿠폰 필터링

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-003 |
| 기능명 | 쿠폰 상태 필터 |
| 설명 | 홈에서 사용가능, 사용완료, 만료 쿠폰을 분리해 표시한다. |

| 출력/결과 | 선택 필터에 맞는 쿠폰 목록 표시 |
| 예외 처리 | 결과가 없으면 empty 마스코트 UI 표시 |

## 5. 쿠폰 검색

## 6. 이미지 선택 및 OCR 정보 추출

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-006 |
| 기능명 | 이미지 기반 자동 입력 |
| 설명 | 갤러리에서 선택한 이미지에서 텍스트, 바코드, QR 정보를 추출해 등록 폼에 반영한다. |
| 입력값 | 갤러리 이미지 파일 |
| 처리 로직 | `image_picker`로 이미지를 선택하고, ML Kit TextRecognizer와 BarcodeScanner를 사용한다. 쿠폰은 코드, 쿠폰명, 브랜드, 카테고리, 날짜 후보를 반영하고, 멤버십은 카드번호와 텍스트 후보를 반영한다. |
| 출력/결과 | 입력 필드 자동 채움 또는 사용자 안내 메시지 |
| 예외 처리 | 이미지가 없거나 인식 결과가 없으면 안내 스낵바 표시. 실시간 카메라 스캔은 제공하지 않는다. |

## 9. 알림 스케줄링

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-009 |
| 기능명 | 쿠폰 만료 로컬 알림 스케줄링 |
| 설명 | 쿠폰 유효기간 기준 D-30, D-7, D-3, D-1, D-DAY, 만료 다음날 알림을 예약한다. |
| 입력값 | Coupon, NotificationSettingsModel |
| 처리 로직 | 사용 완료 또는 만료 쿠폰은 제외한다. 설정 토글이 켜진 타입만 예약한다. 발송 시각은 현재 구현 기준 정오 12시이다. 예약 시 `notification_logs`에 로그도 upsert한다. |
| 출력/결과 | OS 로컬 알림 예약, 최근 알림 인박스 표시 데이터 생성 |
| 예외 처리 | 예약 시간이 현재보다 과거면 예약하지 않는다. 웹에서는 알림 플러그인 호출을 건너뛴다. |

## 10. 알림 설정 토글 연동

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-010 |
| 기능명 | 알림 설정 |
| 설명 | 사용자가 알림 주기별 ON/OFF를 제어한다. |
| 입력값 | 마스터 토글, D-30/D-7/D-3/D-1/당일 토글 |
| 처리 로직 | 설정을 `notification_settings`에 저장하고, 전체 예약 알림을 재스케줄한다. Android 13 이상은 시스템 권한, Android 12 이하는 앱 내부 동의 다이얼로그를 사용한다. |
| 출력/결과 | 알림 예약 상태 변경 |
| 예외 처리 | 마스터 OFF면 전체 예약 알림 취소. 권한 또는 내부 동의가 없으면 ON 처리 중단. |

## 11. 알림 클릭 라우팅

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-011 |
| 기능명 | 알림 클릭 후 쿠폰 상세 이동 |
| 설명 | OS 알림 또는 알림 리스트 항목을 클릭하면 해당 쿠폰 상세로 이동한다. |
| 입력값 | payload (`couponId|type` 또는 과거 호환 `couponId`) |
| 처리 로직 | payload를 파싱하고 쿠폰을 찾은 뒤 알림 로그를 읽음 처리한다. 네비게이터가 준비되지 않은 시점에는 payload를 pending 상태로 보관하고 첫 프레임 이후 또는 앱 resume 시 다시 처리한다. 네비게이션 스택은 홈 다음 쿠폰 상세가 되도록 구성한다. |
| 출력/결과 | CouponDetailScreen 표시 |
| 예외 처리 | 쿠폰이 없으면 이동하지 않는다. 앱 아이콘 재실행 시 직전 알림 상세로 재이동하지 않도록 상태를 초기화한다. |

## 12. D-DAY 계산

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-012 |
| 기능명 | D-DAY 계산 |
| 설명 | 쿠폰 만료일까지 남은 일수를 계산한다. |
| 입력값 | 유효기간 날짜 |
| 처리 로직 | 현재 날짜의 자정과 만료일 자정의 차이를 일수로 계산한다. `0`은 D-DAY, 음수는 만료 상태로 본다. |
| 출력/결과 | D-DAY 숫자, 상태 뱃지, 필터 상태 |
| 예외 처리 | 날짜 파싱 실패 시 상세 모델에서는 만료 아님으로 처리하고, 등록 시에는 입력 검증을 우선한다. |

## 13. 설정 화면 테스트 데이터 생성

| 항목 | 내용 |
|---|---|
| 기능 ID | FN-013 |
| 기능명 | 테스트 쿠폰/가상 멤버십 생성 |
| 설명 | 스토어 스크린샷 촬영과 QA를 위해 설정 화면에서 데모 데이터를 생성한다. |
| 입력값 | 테스트 쿠폰 추가 버튼, 가상 멤버십 추가 버튼 |
| 처리 로직 | 테스트 쿠폰은 D-30/D-7/D-3/D-1/D-DAY/만료/사용완료 상태로 저장한다. 가상 멤버십은 OK캐시백, 해피포인트, L.POINT, CJ ONE을 고정 ID로 upsert한다. |
| 출력/결과 | 쿠폰 리스트와 멤버십 리스트에 데모 데이터 표시 |
| 예외 처리 | 고정 ID를 사용해 버튼을 여러 번 눌러도 동일 항목을 갱신하고 중복 생성하지 않는다. |
