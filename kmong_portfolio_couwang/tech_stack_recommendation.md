# 크몽 직업 / 기술 정보 추천

## 분야: 기획

**선택할 기술:** 서비스 기획, IA, 화면 설계, 스토리보드  
**선택 이유:** 쿠왕은 아이디어 수준의 문제를 기능 정의, 화면 흐름, 등록/알림 정책, 스토어 제출 문서까지 연결한 프로젝트입니다. 기획 역량을 보여주기에 적합합니다.  
**주의할 점:** “PM 총괄”처럼 조직 단위 역할을 과장하기보다, 실제 수행한 서비스 기획 / 화면 구조화 중심으로 적는 것이 안전합니다.

## 분야: 프론트엔드

**선택할 기술:** Flutter, Dart  
**선택 이유:** 앱 전체가 Flutter / Dart 기반으로 구현되어 있으며, 홈, 등록, 상세, 알림, 설정, 멤버십 등 주요 화면이 공통 코드 구조로 관리됩니다.  
**주의할 점:** “웹 프론트엔드 프레임워크 경험”으로 확장해서 쓰기보다, 모바일 UI 구현 중심의 Flutter 경험으로 적는 편이 정확합니다.

## 분야: 모바일

**선택할 기술:** Android, iOS  
**선택 이유:** Android는 실제 공개 링크 기준으로 확인 가능하고, iOS는 Flutter 기반 구조와 빌드 설정, 배포 대응 문서가 준비된 상태입니다.  
**주의할 점:** Android 공개 성과와 iOS 대응 가능성을 구분해서 적어야 합니다. “iOS 출시 완료”처럼 단정적인 표현은 피하는 것이 좋습니다.

## 분야: 백엔드

**선택할 기술:** Firebase  
**선택 이유:** 프로젝트에 Firebase Core / Analytics / Crashlytics가 실제 연결되어 있습니다.  
**주의할 점:** 이 프로젝트는 Firebase를 운영 분석과 안정화 도구로 활용한 사례에 가깝습니다. 일반적인 회원 시스템 / 서버 API 백엔드 구축 경험처럼 보이게 쓰지 않는 것이 안전합니다.

## 분야: 개발 환경 구축

**선택할 기술:** Git, GitHub, Android Studio, VS Code, Xcode  
**선택 이유:** 버전 관리, Android 빌드, Flutter 개발, iOS 빌드 설정 및 배포 준비 흐름을 모두 다뤘습니다.  
**주의할 점:** Xcode는 iOS 빌드 및 설정 대응 경험으로 적고, “iOS 상용 운영 경험”과는 구분하는 것이 좋습니다.

## 분야: 기타

**선택할 기술:** Google Play Console  
**선택 이유:** Android 공개 기준으로 스토어 등록정보, 출시노트, 공개 링크까지 정리된 프로젝트입니다.  
**주의할 점:** App Store Connect는 별도 공개 성과가 확인되지 않는다면 “심사/배포 대응 준비” 수준으로만 설명하는 것이 보수적입니다.
